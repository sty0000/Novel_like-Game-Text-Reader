# 09-结果固化指南

## 目标

确保每轮 TTS 实验的结果不会只留在临时文件里，而是能回到文档中，形成可追溯的选型依据。

## 固化原则

1. **命令必须保留**：没有命令就无法复现。
2. **输入路径必须保留**：包括文本、参考音频、配置。
3. **输出路径必须保留**：音频不一定提交，但路径和文件名规则必须记录。
4. **主观评价必须结构化**：使用统一评分表。
5. **失败也要记录**：失败样本和错误信息是重要结论。

## 每轮实验后更新哪些文档

| 场景 | 更新文档 |
| --- | --- |
| Edge TTS smoke | `04-Edge-TTS-短期闭环.md`、`08-实验记录模板.md` 的复制记录 |
| GPT-SoVITS 实验 | `05-GPT-SoVITS-核心路线.md`、实验记录 |
| 候选模型横评 | `06-候选模型对比-CosyVoice-IndexTTS-FishSpeech.md` |
| 选型变化 | `01-模型选型结论.md`、`00-总览与路线图.md` |
| 阶段总结 | `README.md` 和报告资产 |

## 推荐目录

```text
investigation/docs/generated/
  env/
  corpus/
  edge_tts/
  gpt_sovits/
  comparison/
  report/
```

## 固化流程

1. 运行脚本生成 JSON/CSV/Markdown。
2. 检查生成文件是否包含命令、输入、输出路径。
3. 将摘要复制到对应模型文档。
4. 将评分表复制到实验记录。
5. 如结论发生变化，更新 `01-模型选型结论.md`。
6. 用 `05_generate_report_assets.py` 生成总报告。

## 不应提交的内容

- 模型权重。
- 大体积音频文件。
- 未授权私人语音样本。
- 包含隐私路径或 token 的日志。

## 可提交的内容

- 模板。
- 小型 JSON/CSV 摘要。
- Markdown 报告。
- HTML 阅读版。
- 不含敏感信息的环境摘要。
