# 03-执行手册

本手册是中文执行文档，目标是让后续实验可以按步骤复现。

## 0. 前置约定

所有生成文件默认放入：

```text
investigation/docs/generated/
```

该目录用于保存脚本生成的环境信息、样本文本、命令清单、模板和报告资产。不要在仓库中提交模型权重、大音频文件或私人语音样本。

## 1. 收集环境信息

目的：记录本机 Python、系统、可用命令，为后续复现实验提供背景。

命令：

```bash
python investigation/scripts/00_env_check.py --output-dir investigation/docs/generated/env
```

输出：

- `investigation/docs/generated/env/env.json`
- `investigation/docs/generated/env/env.md`

固化方式：把 `env.md` 中的摘要复制到本轮实验记录，或在 PR 描述中说明环境文件已生成。

## 2. 生成 benchmark corpus

目的：创建所有模型共用的测试文本，避免模型之间使用不同输入导致比较失真。

命令：

```bash
python investigation/scripts/01_build_benchmark_corpus.py --output-dir investigation/docs/generated/corpus
```

输出：

- `benchmark_cases.jsonl`
- `benchmark_cases.csv`
- `benchmark_cases.md`

验证：打开 `benchmark_cases.md`，确认包含旁白、对白、情绪句、术语、中英混读。

## 3. 生成 Edge TTS 执行计划

目的：先打通工程闭环。该脚本不直接调用在线 TTS，只生成命令和记录模板。

命令：

```bash
python investigation/scripts/02_plan_edge_tts_run.py --corpus investigation/docs/generated/corpus/benchmark_cases.jsonl --output-dir investigation/docs/generated/edge_tts
```

输出：

- `edge_tts_commands.md`
- `edge_tts_run_template.json`

执行后续真实合成时，把命令清单中的命令逐条运行，并把结果填回模板。

## 4. 生成 GPT-SoVITS 主线执行规划

目的：为特定角色声音训练/复刻准备样本目录、voice map 和记录模板。

命令：

```bash
python investigation/scripts/03_plan_gpt_sovits_run.py --output-dir investigation/docs/generated/gpt_sovits
```

输出：

- `gpt_sovits_voice_map.template.json`
- `gpt_sovits_sample_manifest.template.csv`
- `gpt_sovits_runbook.md`

保存内容：每个角色的参考音频路径、文本、时长、质量备注、推理输出路径。

## 5. 生成候选模型对比计划

目的：为 CosyVoice、IndexTTS、Fish Speech 生成统一对比表和执行框架。

命令：

```bash
python investigation/scripts/04_plan_model_comparison.py --output-dir investigation/docs/generated/comparison
```

输出：

- `model_comparison_matrix.csv`
- `model_comparison_plan.md`

## 6. 生成报告资产

目的：把前面生成的环境、corpus、执行计划和对比表汇总成一份可读报告。

命令：

```bash
python investigation/scripts/05_generate_report_assets.py --generated-dir investigation/docs/generated --output-dir investigation/docs/generated/report
```

输出：

- `investigation_summary.md`
- `investigation_summary.html`

## 7. 固化到文档

每次实际运行模型后，应更新：

- `08-实验记录模板.md` 的复制记录。
- `09-结果固化指南.md` 中列出的结论摘要。
- 对应模型路线文档中的“已验证情况”。

建议每轮实验至少记录：

```text
日期、模型、版本、输入文件、命令、输出路径、主观评分、问题、下一步
```

## 8. 失败处理

- 脚本找不到输入：先确认路径是否为仓库根目录相对路径。
- 生成目录不存在：脚本会自动创建；如果失败，检查权限。
- TTS 实际合成失败：不要直接改 benchmark corpus，先记录错误和模型环境。
- 结果太大：不要提交音频大文件，保留路径和摘要即可。
