# 10-情绪标注与多角色演绎方案

## 目标

本方案用于解决剧情朗读中“不要总是 `xxx 说`”的问题：在结构化剧情片段中增加情绪、强度、说话方式和证据字段，让后续 TTS 引擎可以根据角色与情绪选择不同表达方式。

核心原则：**先结构化标注，再生成朗读文本或 TTS 参数，不直接改写原始剧情正文。**

## 处理层次

```text
PRTS raw source
  → 剧情片段 JSONL
  → 情绪标注 emotion / intensity / evidence
  → 说话方式 speech_verb
  → 角色 voice map + 情绪 voice style
  → TTS 输入文本 / prompt / reference audio
  → 分段音频
```

## 推荐 JSONL 字段

原始片段：

```json
{
  "segment_id": 1,
  "speaker": "阿米娅",
  "role": "dialogue",
  "text": "博士，博士！你终于醒了。"
}
```

情绪标注后：

```json
{
  "segment_id": 1,
  "speaker": "阿米娅",
  "role": "dialogue",
  "text": "博士，博士！你终于醒了。",
  "emotion": "urgent",
  "intensity": 0.8,
  "speech_verb": "急切地说",
  "narration_prefix": "阿米娅急切地说：",
  "evidence": ["重复称呼", "感叹号"]
}
```

## 情绪标签第一版

先不要设计太多标签，建议从 12 个开始：

| emotion | 中文含义 | 默认表达 |
| --- | --- | --- |
| neutral | 中性 | 说 |
| urgent | 急切 | 急切地说 |
| angry | 生气 | 生气地说 |
| sad | 低落 | 低落地说 |
| fearful | 害怕 | 害怕地说 |
| surprised | 惊讶 | 惊讶地说 |
| hesitant | 迟疑 | 迟疑地说 |
| whisper | 低声 | 低声说 |
| determined | 坚定 | 坚定地说 |
| apologetic | 歉疚 | 歉疚地说 |
| confused | 困惑 | 困惑地说 |
| relieved | 放松 | 松了一口气说 |

## 判断情绪的方法

### 1. 规则启发式

第一版优先使用规则，理由是可解释、可复现、便于人工修正。

可用信号：

| 信号 | 倾向情绪 |
| --- | --- |
| `！`、重复称呼 | urgent / surprised |
| `？`、`？！` | confused / surprised |
| `……`、`......` | hesitant / sad |
| “快”“小心”“住手”“别过来” | urgent / fearful |
| “可恶”“混蛋” | angry |
| “抱歉”“对不起” | apologetic |
| “太好了”“谢谢” | relieved |
| `[Delay]` | hesitant / tense |

### 2. LLM 辅助标注

规则无法判断时，可以把上下文交给 LLM，但只让它输出结构化标签，不让它改剧情正文。

建议输出：

```json
{
  "emotion": "anxious",
  "intensity": 0.65,
  "speech_verb": "不安地说",
  "reason": "文本包含省略号和请求语气，前文有道歉。"
}
```

### 3. 人工 override

剧情情绪具有主观性，必须允许人工覆盖：

```json
{
  "segment_id": 104,
  "emotion_override": "determined",
  "speech_verb_override": "坚定地说"
}
```

最终优先级：

```text
人工 override > LLM 标注 > 规则标注 > neutral
```

## 何时添加“激动地说”这类前缀

不要每一句都加修饰。推荐规则：

添加前缀：

- 说话人切换。
- 情绪变化。
- `intensity >= 0.6`。
- 当前句是角色首次强情绪发言。

不添加修饰：

- `intensity < 0.4`。
- 同一角色连续多句且情绪不变。
- 文本本身已经明显表达情绪。

## 不同 TTS 引擎的落地方式

### Edge TTS

Edge TTS 不支持训练特定声音，情绪只能通过文本前缀、语速、音量和标点弱模拟：

```text
阿米娅急切地说：博士，博士！你终于醒了。
```

可配合：

- urgent：`--rate +10%`
- sad：`--rate -10%`
- angry：`--volume +10%`

### GPT-SoVITS

GPT-SoVITS 更适合通过 reference audio / prompt audio 控制情绪和说话风格。建议为重要角色准备不同情绪参考音频：

```text
voices/amiya/
  neutral/ref.wav
  urgent/ref.wav
  sad/ref.wav
```

voice map 示例：

```json
{
  "阿米娅": {
    "neutral": "voices/amiya/neutral/ref.wav",
    "urgent": "voices/amiya/urgent/ref.wav",
    "sad": "voices/amiya/sad/ref.wav"
  }
}
```

### CosyVoice / IndexTTS2

这类模型更适合后续测试 instruction / emotion control：

```text
用紧张、急切的语气说这句话<|endofprompt|>
```

IndexTTS2 方向重点关注 emotion control、zero-shot voice cloning 和 emotion-speaker disentanglement。


## 后续集成建议

1. 在 `parse_story.py` 之后增加 emotion tagging 步骤。
2. 保持原始 `text` 不变，仅新增 emotion 字段。
3. TTS adapter 根据不同 engine 决定如何使用 emotion：
   - Edge TTS：文本前缀 + rate/volume。
   - GPT-SoVITS：选择情绪 reference audio。
   - CosyVoice / IndexTTS2：instruction 或 emotion 参数。
4. 人工校正文件单独保存，不覆盖自动标注结果。

## 参考方向

- GPT-SoVITS：重点测试 reference audio 对语速、风格和情绪的影响。
- CosyVoice：重点测试 instruct / prompt 控制。
- IndexTTS2：重点测试 emotion control 和音色/情绪解耦。
