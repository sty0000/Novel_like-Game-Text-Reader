# 00-总览与路线图

## 项目目标

当前项目已经能够从 PRTS wiki 获取《明日方舟》剧情文本，并逐步转向“剧情语料预处理 + TTS 朗读”。本轮 investigation 的目标不是立刻训练最终模型，而是建立一套可复现、可记录、可横向比较的 TTS 模型调研流程。

## 需求复述

目标需求：

- 每个声音只有多段一句话音频和对应文本。
- 总语音长度预计不超过约 1 分钟。
- 希望训练或复刻一个特定声音。
- 最终能够让多个声音共同演绎剧情对白。

这个需求更接近 **few-shot voice cloning / zero-shot voice cloning / 少样本微调**，不适合从零训练完整 TTS 模型。

## 总体路线

```text
剧情源码
  → 结构化 JSONL 片段
  → 角色到音色映射
  → TTS 引擎调度
  → 分段音频
  → 拼接与质量检查
  → 记录固化到文档
```

## 阶段规划

### 阶段 1：Edge TTS 工程闭环

目的：验证脚本结构、输入输出目录、分段音频落盘、执行记录是否可用。  
不解决：特定声音训练、角色声线复刻。

产物：

- benchmark 文本集
- Edge TTS 命令清单
- 运行记录模板
- 输出路径约定

### 阶段 2：GPT-SoVITS 主线验证

目的：验证“约 1 分钟以内语音文本对照样本”是否能得到可用角色音色。  
重点：样本质量、文本对齐、角色 voice profile、推理稳定性。

产物：

- 角色样本目录规范
- voice map 模板
- GPT-SoVITS 执行手册
- 实验记录表

### 阶段 3：候选模型横评

候选：CosyVoice、IndexTTS、Fish Speech。  
目的：在同一文本集和同一记录格式下比较 zero-shot / voice cloning 方案。

产物：

- 模型对比矩阵
- 统一评分表
- 对比计划文档

### 阶段 4：结论固化

每轮实验必须把环境、输入、命令、输出、主观评价和下一步建议写回文档，避免只留下临时文件。

## 里程碑

| 里程碑 | 完成标准 |
| --- | --- |
| M1 文档与脚本框架 | `investigation/docs` 和 `investigation/scripts` 完整存在 |
| M2 benchmark corpus | 生成 JSONL/CSV/Markdown 三种格式 |
| M3 Edge TTS 闭环 | 能得到可执行命令清单和记录模板 |
| M4 GPT-SoVITS 规划 | 有角色样本规范、voice map、runbook |
