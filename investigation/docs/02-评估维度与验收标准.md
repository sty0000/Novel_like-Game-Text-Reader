# 02-评估维度与验收标准

## 目标

统一所有 TTS 模型的评估口径，避免每个模型用不同文本、不同记录方式、不同主观标准得出不可比较的结论。

## 评分维度

每项 1–5 分：

| 维度 | 说明 |
| --- | --- |
| 中文发音准确度 | 普通话、声调、断句、常见词读音 |
| 明日方舟术语 | 角色名、地名、组织名、中英混读 |
| 音色相似度 | 与参考声音的相似程度 |
| 音色稳定性 | 多句连续生成时是否漂移 |
| 情绪表达 | 疑问、惊讶、紧张、低声、旁白等 |
| 长句稳定性 | 长文本是否吞字、重复、断裂 |
| 多角色调度 | 是否便于按 speaker 切换 voice |
| 部署复杂度 | 安装、模型下载、环境冲突 |
| 推理成本 | 是否需要 GPU、速度、显存 |
| 可复现性 | 参数、版本、输入输出是否可追踪 |

## 必测文本类型

benchmark corpus 必须覆盖：

1. 短对白：2–8 秒内可读完。
2. 长对白：包含多个标点和语气变化。
3. 旁白：稳定叙述，不要求强情绪。
4. 情绪句：惊讶、压低声音、急促、迟疑。
5. 明日方舟术语：博士、阿米娅、罗德岛、源石、切尔诺伯格。
6. 中英混读：PRTS、Rhodes Island、Doctor。
7. 标点压力：省略号、破折号、感叹号、问号。

## 通过标准

### Edge TTS

- 能生成音频文件。
- 命令和输出路径可复现。
- 不要求音色相似度。

### GPT-SoVITS

- 至少一个角色样本可以完成推理。
- 输出音色与参考音频有可感知相似度。
- 短句不出现明显吞字、乱码、重复。
- 所有参数、样本路径和输出路径记录完整。

### 候选模型

- 可以在同一 benchmark corpus 上运行。
- 输出目录结构一致。
- 评分表可横向比较。

## 不通过条件

- 输出音频无法播放。
- 记录缺少命令或输入路径，无法复现。
- 生成内容与输入文本严重不符。
- 需要提交模型权重、音频大文件或敏感数据才能复现。
